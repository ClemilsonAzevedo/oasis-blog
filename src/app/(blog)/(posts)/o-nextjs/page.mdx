>A pasta pages é onde englobamos, a pasta API, as páginas da aplicação e ela pode estar ou não dentro de uma pasta src

## Rotas de uma aplicação Nextjs

No nextjs nós temos o **File-System Routing** ou seja, apenas criando um arquivo com o nome desejado e dentro ter um componente exportado como default, já cria uma rota.
Embora também podemos criar uma pasta dentro de pages com o nome da rota e nomear o arquivo dentro dela de index.tsx

Se por acaso eu quiser uma rota dinamica, como passar um id em uma rota de produtos para selecionar um produto, em vez de index.tsx eu poderia criar um `[id].tsx`, assim podendo passar qualquer id na rota.

Se por acaso eu quiser pegar parametros da rota do Next posso usar do next/router o hook useRouter.


## Document no Next

Para configurar coisas relacionadas ao HTML no Next, devemos criar um arquivo document dentro da pasta pages e:
```tsx
//_document.tsx
import { Html, Head, Main, NextScript } from 'next/document'

export default function Document(){

return(
	<Html>
		<Head>
			//Link das fonts aqui
		</Head>
		<body>
			<Main />
			<NextScript />
		</body>
	</Html>
)
}
```

## Estilos

Não devemos colocar a pasta styles dentro de Pages, melhor criar uma pasta pages dentro de styles e estilizar cada uma das páginas.
